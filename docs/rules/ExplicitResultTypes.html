ExplicitResultTypes

⚠️ Experimental

This rewrite inserts the inferred type from the compiler for implicit definitions that are missing an explicit result type.

For example,

// before
implicit val tt = liftedType

// after
implicit val tt: TypedType[R] = liftedType

But for local implicit val and def it's not needed and will not be added by default.

def foo = {
  // this is OK
  implicit val localImplicit = 2
}

Configuration

This rule has several different options.

NameTypeDescription
memberKindList[MemberKind]Enable/disable this rule for defs, vals or vars.
memberVisibilityList[MemberVisibility]Enable/disable this rule for private/protected members.
skipSimpleDefinitionsBooleanIf false, insert explicit result types even for simple definitions like `val x = 2`
skipLocalImplicitsBooleanIf false, insert explicit result types even for locally defined implicit vals
unsafeShortenNamesBooleanIf true, does a best-effort at inserting short names and add missing imports. WARNING. This feature is currently implemented in a naive way and it contains many bugs.
fatalWarningsBooleanIf true, report and fail unexpected errors. If false, silently ignore errors to produce an explicit result type.

Defaults:

ExplicitResultTypes.memberKind = []
ExplicitResultTypes.memberVisibility = []
ExplicitResultTypes.skipSimpleDefinitions = true
ExplicitResultTypes.skipLocalImplicits = true
ExplicitResultTypes.unsafeShortenNames = false
ExplicitResultTypes.fatalWarnings = false

Examples

ExplicitResultTypes.memberKind = [Def, Val, Var]
ExplicitResultTypes.memberVisibility = [Public, Protected]

Known limitations

  • [#324](https://github.com/{{ site.githubOwner }}/{{ site.githubRepo }}/issues/324)

  • This rewrite inserts fully qualified names by default, even if the short name is available in scope. Blocked by scalameta/scalameta#1075.

  • Inserts type annotations only for implicit definitions by default. To configure the rule to run on non-implicit definitions

  • The rewrite may insert conflicting names in scope. This is especially true with unsafeShortenNames=true, but may happen by default for path dependent types.
